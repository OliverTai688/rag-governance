---
本提案所設計之三種導入層級，並非三種不同系統，而是**同一套治理架構在不同成熟度階段的導入深度差異**。三個方案皆包含完整案件資料庫（SSOT）建置、專案節奏設計與基本治理框架，其差異主要在於「流程建模範圍」、「風險提醒層級」與「策略分析能力」。

換言之，三本提案的三種方案設計，對應組織治理成熟度的三個階段：

穩定化（Stabilization）、優化化（Optimization）、預測化（Predictive Governance）。

其核心差異並非功能數量，而是決策支撐能力的層級。

---

## **共同基礎（所有方案皆包含）**

無論選擇哪一方案，皆會完成以下核心結構：

* 建立完整案件資料庫（整合既有資料與 LINE 端口資訊）

* 建立 SSOT（Single Source of Truth）

* 建立專案節奏制度

* 建立基本治理可視性

* 建立固定教育訓練與陪跑期

因此，選擇方案時不必擔心「是否會導入完整資料」，因為資料層是所有方案的基礎。真正的差異，在於治理能力的深度。

# 維護與顧問方案（AI 內含）

### **基礎維護：適合「已導入但暫無擴張需求」**

基礎維護方案的核心目的，在於確保已導入之治理結構能穩定運作，並提供日常技術維護與基本 AI 輔助功能。此層級適合案件量相對穩定、組織結構清楚且暫無擴張規劃的事務所。透過基礎回覆優化與節奏提醒機制，可降低日常溝通負擔，並確保案件管理維持在可控狀態。然而，此層級不包含主動式風險報告與顧問會議支持，治理能力的優化將主要仰賴內部自行推進。因此，其定位在於「維持穩定」，而非「推動成長」。

---

### **成長顧問：適合「希望降低管理依賴個人經驗」**

成長顧問方案的核心價值，在於將治理結構由「可運作」提升至「可優化」。在此層級中，系統將每月生成專案節奏與風險分析報告，並結合客戶輪廓的季度更新，使管理者能看見承辦負荷分布、退件趨勢與流程瓶頸。搭配固定顧問會議，組織不僅能觀測問題，更能逐步調整流程與配置策略。此方案特別適合案件量持續增加、團隊人數擴張或希望降低對個人經驗依賴的事務所。其本質並非增加工具，而是建立「持續改善機制」，讓治理能力隨時間深化，而非停留在導入當下。

---

### **自動化強化：適合「準備擴張或標準化複製的組織」**

自動化強化方案屬於高成熟度階段，強調模型判斷型風險優先排序與全流程分析能力。系統將綜合人力負荷、案件密度、急件比例與歷史趨勢進行預測性分析，並提供策略建議簡報，協助管理者進行中長期資源規劃。此層級適合已有擴張計畫、希望建立可複製管理模型，或正面臨高成長壓力的事務所。然而，若組織目前仍在穩定流程與建立基礎節奏階段，直接進入預測型治理可能會超出實際需求。因此，此方案更適合作為中長期升級目標，而非初期必選項。

---

### **建議選擇方向**

綜合目前地政事務所之案件成長情境與管理需求，本提案建議導入後的前6個月優先採用「成長顧問」方案。此層級能在不過度複雜化系統的前提下，建立穩定的優化節奏與風險觀測機制，使組織由經驗型管理逐步轉為資料驅動型治理。同時保留未來升級至自動化強化層級的彈性。換言之，成長顧問是兼顧穩定與發展的平衡點，既能回應當前需求，也能為未來擴張預作準備。

| 項目 | 基礎維護 | 成長顧問 | 自動化強化 |
| ----- | ----- | ----- | ----- |
| **月費** | 1,000 NTD | 3,000 NTD | 8,000 NTD |
| 技術維護 | 含 | 含 | 含 |
| 系統更新 | 含 | 含 | 含 |
| 每月資料備份 | 含 | 含 | 含 |
| AI 回覆建議 | 基礎回覆優化  | 案件語境分析＋回覆策略建議 | 全流程自動分析＋策略生成 |
| AI 用量 (tokens) | 150,000  | 400,000 | 1,000,000  |
| 客戶輪廓更新 | 無 | 基礎輪廓建構（季度更新） | 動態自動更新（每月調整） |
| 專案節奏與風險報告 | 無 | 每月自動生成 | 每月生成＋優化建議簡報 |
| 節奏顧問會議 | 無 | 每月 1 次 | 每月 1 次（含優化檢視） |
| 自動提醒機制 | 基礎案件節奏提醒 | 基礎案件節奏提醒＋基礎風險提醒 | 基礎案件節奏提醒＋風險優先排序（產業風險結構每月調整） |
| 適合對象 | 穩定使用者 | 持續優化型 | 高成長轉型型 |

註：轉型自動化層級因涉及模型優化與持續調整，建議至少搭配成長顧問以上之維護方案，以確保模型穩定與風險預測準確度。  
註：所有方案皆導入全案件資料，差異在於流程模型建構深度與提醒機制層級。

## **AI 能力擴充模組（彈性增購）**

為避免一次性升級整體方案，系統提供 AI 使用量與分析能力之彈性擴充機制。  
**適用情境：**當案件量提升、AI 回覆需求增加、或需進行跨專案深度分析時。

### **Token 擴充方案**

| 擴充單位 | 費用 |
| ----- | ----- |
| 100,000 tokens | 600 NTD |
| 300,000 tokens | 1,500 NTD |
| 1,000,000 tokens | 4,000 NTD |

可隨時追加，當月有效，不影響原方案層級。

註：AI Token 為運算資源成本，依實際使用量計算，系統將於 80% 使用量時自動提醒。
